Apr 1, 2015

原文：http://linux.im/2015/03/29/alictf-2015-xss400.html

根据文章自己测试了下，很有意思。

http://1de28830f09a4b1b.alictf.com/pet.php?id=test&type=cat 访问，显示一张图片。

查看源代码，比较简短。

<script>
    if(!window.chrome){
        alert('\u8bf7\u4f7f\u7528\u0043\u0068\u0072\u006f\u006d\u0065\u6d4f\u89c8\u5668\u6d4f\u89c8\u672c\u9875');
        location.href = 'https://www.google.com/chrome/';
    }
</script>
<script src="secWrite.js"></script>
<script>var imageData = {name: "Oreo"}</script>
</head>
<body>
    <div id="console"></div>
    <img id="test" src="cat.jpg" name="cat" onload="loadSucc()" onerror="loadErr()">
    <script>
        function loadSucc(){
            document.write('This is '+imageData.name)    
        }

        function loadErr(){
            document.write('Load '+imageData.name+' Fail!')    
        }

    </script>
</body>
</html>

secWrite.js代码如下：

document.write = function(str){
    document.getElementById('console').innerText = str;
}

根据wirteup，这里用到了好几个技巧。

第一个利用chrome的filte干掉一些js代码。使用

http://1de28830f09a4b1b.alictf.com/pet.php?id=test&type=cat&b=%3Cscript%3Evar%20imageData%20=%20{name:%20%22Oreo%22}%3C/script%3E

可以干掉返回中的

<script>var imageData = {name: "Oreo"}</script>，

但是无法干掉<script src="secWrite.js"></script>，

猜测chrome的filter的机制就是这样的。只处理inline方式的。

然后利用的技巧是由于 img标签的id和name属性是可控的，达到控制imageData.name的目的。这里之前一直觉得或得标签的属性先要选定id，再get属性。

第三个技巧就是干掉secWrite.js.通过使用/?，请求pet.php/secWrite.js失败，从而绕过document.write的hook。

最后一个技巧就是比较常见的编码问题了。js从元素属性过去值，然后写入页面。元素属性可以用html编码。10进制，16进制都可以。
