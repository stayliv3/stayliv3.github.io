phpshe 商城安装过程分析

Mar 24, 2015

http://wooyun.org/bugs/wooyun-2010-062047
phpshe商城之前没有对重装的流程做校验，存在任意重新安装的漏洞。其实对于安装过程存在的风险，好的安全解决方案应该是安装完成之后删除安装文件夹。

分析版本PHPSHE1.3

安装流程可能存在的风险分析：

这里对phpshe进行分析，发现可能存在的风险就是变量覆盖。最后的分析结果是由于extrack使用了前缀，导致变量无法覆盖。

include("{$pe['path_root']}/include/function/global.func.php");

if (get_magic_quotes_gpc()) {
    !empty($_GET) && extract(pe_trim(pe_stripslashes($_GET)), EXTR_PREFIX_ALL, '_g');
    !empty($_POST) && extract(pe_trim(pe_stripslashes($_POST)), EXTR_PREFIX_ALL, '_p');
}
else {
    !empty($_GET) && extract(pe_trim($_GET),EXTR_PREFIX_ALL,'_g');
    // !empty($_GET) && extract(pe_trim($_GET));

    !empty($_POST) && extract(pe_trim($_POST),EXTR_PREFIX_ALL,'_p');
}
var_dump($_GET);
echo $_g_step;
switch ($_g_step) {
    //#####################@ 配置信息 @#####################//
    case 'setting':
        if (is_file("{$pe['path_root']}install/install.lock")) die('PHPSHE商城系统已经安装成功，如需再次安装请删除 ./install/install.lock 文件');

is_file判断文件是否存在。前面使用extract把GET，POST变量变为全局变量。看起来刚好是一个变量覆盖。但是extract使用了EXTR_PREFIX_ALL参数，会给每一个变量添加前缀。所以也就没法覆盖了。看来使用前缀也是一种避免变量覆盖的防御方案。PHP其实内置了很多特性。

测试去掉EXTR_PREFIX_ALL参数，是可以实现覆盖进入下面流程的。

php手册上说is_file在某些平台上判断超过2G的文件会存在问题。YY一下，如果写lock文件可以append方式写入，写到2G会不会导致出现重装的问题？不过PHP能不能写2G的文件也是需要考虑的。
