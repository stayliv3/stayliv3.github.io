#分析记录

thinkphp作为一个框架，在底层上是有做了一些安全工作的。所以寻找SQL的思路就变成了寻找框架上漏掉过滤的点和框架特性上可以直接执行SQL语句的地方。

##特性1：exp表达式

http://www.kancloud.cn/manual/thinkphp/1768

exp特性可以用于数据查询和数据更新。save()方法。

这里从filter_exp函数可以看出，开发者是有考虑到用户如果输入exp，导致sql注入的问题。所以进行了一次过滤。但是过滤的方式过于随意。只是在后面加了个空格。而且全局过滤存在问题。通过http://serverName/index.php?c=Blog&a=archive&year=2013&month[0]=exp&month[1]=sqli 这种方式调用方法不会通过I函数过滤。

function filter_exp(&$value){

    if (in_array(strtolower($value),array('exp','or'))){

        $value .= ' ';

    }
}
